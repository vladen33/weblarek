# Проектная работа "Веб-ларек" (Интернет-магазин «Web-Larёk»)
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

Стек: HTML, SCSS, TS, Vite

Структура проекта:
- src/ — исходные файлы проекта
- src/components/ — папка с JS компонентами
- src/components/base/ — папка с базовым кодом

Важные файлы:
- index.html — HTML-файл главной страницы
- src/types/index.ts — файл с типами
- src/main.ts — точка входа приложения
- src/scss/styles.scss — корневой файл стилей
- src/utils/constants.ts — файл с константами
- src/utils/utils.ts — файл с утилитами

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
- `constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
- `container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
- `render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
- `setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
- `constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
- `baseUrl: string` - базовый адрес сервера  
- `options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
- `get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
- `post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
- `handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
- `_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
- `on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
- `emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
- `trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.


### Основной код

#### Данные
Основные сущности, используеме в приложении: *Товар (Product)* и *Покупатель (Customer)*. 
Для работы с ними использованы интерфейсы:

```
// Интерфейс для реализации сущности Товар (Product)
interface IProduct {
    id: string; // уникальный идентификатор товара, вида "c101ab44-ed99-4a54-990d-47aa2bb4e7d9"
    description: string; // Описание товара
    image: string; // Изображение товара
    title: string; // Наименование товара
    category: string; // Категория товара
    price: number | null; // Цена товара
}

// Тип платежа может быть либо "Картой", либо "Наличными", либо пустое значение, если вид платежа еще не определен 
type TPayment = 'card' | 'cash' | '';

// Интерфейс для реализации сущности Покупатель (Customer)
interface ICustomer {
    payment: TPayment; // Тип платежа
    address: string; // Адрес покупателя
    email: string; // почтовый адрес покупателя
    phone: string; // Контактный телефон покупателя
}
```

#### Модели данных
Управление сущностями осуществляется через классы.

#### Класс Catalog
Предназначен для хранения данных о товарах, которые можно купить в приложении.
В парадигме MVP класс относится к Model - слою данных, отвечающему за хранение и изменение данных.

Конструктор класса Catalog не принимает параметров

Поля класса:
- `productList: IProduct[]` - в свойстве хранится список товаров каталога, полученный с сервера
- `selectedProduct: IProduct | null = null` - в свойстве хранится выбранный товар из каталога

Методы класса:
- `setProductList(arr: IProduct[]): void` - сохраняет в классе список товаров, полученный с сервера
- `getProductList(): IProduct[]` - возвращает сохраненный в классе список товаров
- `getProductById(id: string): IProduct` - возвращает товар из каталога в соответсвии с переданным идентификатором товара    
- `setSelectedProduct(prod: IProduct): void` - сохраняет выбранный товар в поле класса `selectedProduct`.
- `getSelectedProduct(): IProduct | null` - возвращает ранее сохраненный в классе товар

#### Класс Cart
Предназначен для хранения списка товаров, которые покупатель выбрал для покупки.
В парадигме MVP класс относится к Model - слою данных, отвечающему за хранение и изменение данных.

Конструктор класса Cart не принимает параметров

Поля класса:
- `prodList: IProduct[]` - хранит список добавленных в корзину товаров

Методы класса:
- `getProdListFromCart(): IProduct[]` - возвращает список продуктов, находящихся в корзине
- `addProdToCart(prod: IProduct): void` - добавляет выбранный покупателем товар в корзину
- `removeProdFromCart(prod: IProduct): void` - удаляет выбранный покупателем товар из корзины
- `clearCart(): void` - удаляет все товары из корзины
- `getFullPriceOfCart(): number` - возвращает полную суммарную стоимость всех товаров, находящихся в корзине
- `getCountProdInCart(): number` - возвращает общее количество товаров
- `isProdInCart(prod: IProduct): boolean` - выполняет проверку, добавлен ли продукт в корзину. Возвращает true, если продукт находится в корзине.

#### Класс Customer
Предназначен для хранения данных покупателя, которые тот должен указать при оформлении заказа.
В парадигме MVP класс относится к Model - слою данных, отвечающему за хранение и изменение данных.

Конструктор класса Customer не принимает параметров

Поля класса:
- `payment: TPayment` - тип платежа (картой, наличными)
- `address: string` - адрес покупателя
- `email: string` - адрес электронной почты покупателя
- `phone: string` - контактный телефон покупателя

Методы класса:
- `setPayment(payment: TPayment): void` - сохраняет метод платежа (картой, наличными)
- `setAddress(address: string): void` - сохраняет адрес покупателя
- `setEmail(email: string): void` - сохраняет адрес электронной почты покупателя
- `setPhone(phone: string): void` - сохраняет телефон покупателя
- `getPayment(): TPayment` - возвращает метод платежа (картой, наличными)
- `getAddress(): string` - возвращает адрес покупателя
- `getEmail(): string` - возвращает адрес электронной почты покупателя
- `getPhone(): string` - возвращает телефон покупателя
- `getAllFields(): ICustomer` - возвращает объект, содержащий все данные покупателя 
- `clearAllFields(): void` - очищает все данные покупателя в классе
- `validateFields(): Record<string, string>` - возвращает объект, в котором в качестве ключей - названия полей класса, в качестве их значений - тест ошибки

### Слой коммуникации

#### Класс MainApi
Класс MainApi предназначен для отправки и получения данных с сервера через API

Конструктор класса MainApi не принимает параметров

Методы класса:

Используемые интерфейсы:
- `IOrderRequest` - описыват структуру объекта, которой должен соответствовать отправляемый на сервер объект

```
export interface IOrderRequest {

    email: string,
    phone: string,
    address: string,
    total: number,
    items: string[]

```
`IOrderResponse` - описыват структуру объекта, получаемого с сервера 
```
export interface IOrderResponse {

}
```


### Слой представления (View)
#### Класс CardBaseView
Класс CardBaseView - это базовый абстрактный класс для представления карточек товара

Поля класса:
- `titleNode` - DOM-элемент для отображения названия товара
- `priceNode` - DOM-элемент для отображения цены товара
- `id` - идентификатор товара

Методы класса:
- `set title(titleValue: string)` - записывает название товара в DOM-элемент
- `set price(priceValue: string | null)` - записывает цену товара в DOM-элемент (если priceValue равняется null, то запивывает значение «Бесценно»)


#### Класс CardBasketView - элемент товара в корзине
Класс CardBasketView наследуется от класса CardBaseView и определяет отображение товара в корзине

Поля класса:
- `indexElement` - DOM-элемент для указания порядкового номера товара в корзине
- `deleteButtonElement` - DOM-элемент кнопки, которая удаляет товар из корзины

Методы класса:
- `set index(indexValue: string)` - записывает порядковый номер товара в соответствующий DOM-элемент


#### Класс CardCatalogView
Класс CardCatalogView наследуется от класса CardBaseView и определяет отображение товара в каталоге на главной странице

Поля класса:
- `imageNode` - DOM-элемент для показа изображения товара
- `categoryNode` - DOM-элемент для показа категории товара

Методы класса:
- `set category(categoryName: string)` - записывает название категории товара в DOM-элемент
- `set image(imageName: string)` - записывает путь к изображению товара товара в DOM-элемент
- `render(data?: Partial<T>)` - подготавливает данные карточки к отображению


#### Класс CardPreviewView
Класс CardPreviewView наследуется от класса CardCatalogView и определяет отображение карточки товара с полным набором данных

Поля класса:
- `textNode` - DOM-элемент для вывода описания товара
- `buttonNode` - кнопка для добавления или удаления товара из корзины
- `inBasket` - признак, добавлен ли товар в корзину

Методы класса:
- `set text(textValue: string)` - записывает описание товара в DOM-элемент
- `render(data: IProduct, inBasket = false)` - подготавливает данные карточки к отображению


#### Класс HeaderView
Класс HeaderView предназначен для управления отображением счетчика товаров в корзине и доступа к работе с корзиной товаров

Поля класса:

Методы класса:

### Презентер (обработчики событий)
#### Для управления элементами приложения используется брокер событий:
- `catalog:change` - перерисовывает каталога на главной странице при любом его изменении
- `product:show` - при клике по карточке в каталоге выводится модальное окно с карточкой подробного описания товара
- `basket:product-add` - при нажатии кнопки в карточке товара, добавляет выбранный товар в корзину
- `basket:product-delete` - при нажатии кнопки удаления товара в модальном окне вывода содержимого корзины, удаляет соответствующий товар
- `basket:update` - при любом изменении в модели данных корзины производится обновление представления корзины
- `basket:open` - открывает корзину и выводит содержимое корзины в модальном окне
- `basket:open-order-form` - открывает форму для указания параметров заказа (тип платежа и адрес доставки)
- `basket:open-contacts-form` - открывает форму для указания контактных данных (email и номер телефона)
- `customer-model:update` - обновляет модель данных покупателя
- `customer-model:has-updated` - актуализирует представления форм заказа и контактов на основании модели данных покупателя
- `basket:try-send-order` - выполняется попытка отправить данные заказа на сервер. В случае успеха вызывается 
- `basket:success` - в случае успешной отправки данных заказа на сервер и получения успешного ответа выводится модальное окно об успешном оформлении заказа
- `basket:success-close` - закрытие окна об успешном оформлении заказа
